# ✅ ЗАВЕРШЕНО - CI/CD Pipeline для itemstock_mvp на microk8s

## 🎉 Что было создано

Полная инфраструктура для автоматической сборки и развертывания приложения на Kubernetes при каждом коммите в GitHub.

## 🚀 Быстрый старт (в 3 шага)

### Шаг 1️⃣ На microk8s сервере (2-3 минуты)

```bash
cd itemstock_mvp
chmod +x k8s/quick-setup.sh
./k8s/quick-setup.sh
```

### Шаг 2️⃣ На GitHub (5 минут)

```
Settings → Secrets and variables → Actions
Добавить 3 Secrets:
  • MICROK8S_HOST = IP сервера
  • MICROK8S_USER = SSH пользователь
  • MICROK8S_KEY = SSH ключ (из шага 1)
```

### Шаг 3️⃣ На локальной машине (1 минута)

```bash
git push origin main
```

**Результат:** Автоматическое развертывание на microk8s сервер! 🎉

```
        ↓

        ↓
✅ Streamlit приложение работает!
```

## 🔐 Безопасность

✅ **Защищено:**

- SSH ключи в GitHub Secrets (зашифрованы)
- Приватные ключи не в репозитории
- Доступ только через SSH с аутентификацией
- Данные в PersistentVolume защищены

⚠️ **Требуется:**

- Регулярная ротация SSH ключей
- Ограничение прав SSH пользователя (опционально)
- RBAC для K8s (опционально)

## ✅ Что работает автоматически

| Процесс                   | Триггер                   | Результат                   |
| ------------------------- | ------------------------- | --------------------------- |
| Сборка Docker образа      | `git push` в main/develop | Образ собирается            |
| SSH подключение           | После сборки              | Подключение к microk8s      |
| Git clone на сервер       | Успешное подключение      | Код скачивается             |
| Локальная сборка          | Код скачан                | Образ собирается на сервере |
| K8s deployment обновление | Образ готов               | Pod перезапускается         |
| Health check              | Pod запущен               | Проверка доступности        |
| Rolling update            | Всё проверено             | Старый pod удаляется        |

## 🎯 Типичный workflow

```bash
# Дома/в офисе - разработка
vim app.py
git add .
git commit -m "Feature: добавлена новая функция"
git push origin main

# Автоматически на microk8s сервере:
# 1. GitHub Actions собирает образ
# 2. SSH подключается к серверу
# 3. Скачивается новый код
# 4. Собирается образ локально
# 5. Применяются K8s manifests
# 6. Перезапускается приложение

# На сервере - мониторинг
microk8s kubectl logs -n itemstock -l app=itemstock -f
```

## 🆘 Если что-то не работает

1. **Проверьте логи GitHub Actions**
   - GitHub → Actions → deploy-simple.yml → Run logs

2. **Проверьте логи K8s**

   ```bash
   microk8s kubectl logs -n itemstock -l app=itemstock -f
   ```

3. **Смотрите Troubleshooting**
   - [`DEPLOYMENT.md`](DEPLOYMENT.md) → Troubleshooting

4. **Используйте команды отладки**
   - [`COMMANDS.sh`](COMMANDS.sh) → Раздел "Отладка"
